# PathOps Documents

This repository contains the guiding documents for **PathOps**.

- The manifesto and principles define *why PathOps exists* and *what it optimizes for*.
- These documents are versioned, reviewed via pull requests, and intended to be referenced by:
  - the main code repositories
  - the website (manifesto.pathops.io)
  - community discussions

## Documents

- **[manifesto.md](./manifesto.md)** — the “why” and the core promise
- **[principles.md](./principles.md)** — non-negotiable design principles
- **[what-pathops-is-not.md](./what-pathops-is-not.md)** — boundaries and anti-goals
- **[glossary.md](./glossary.md)** — shared terminology
- **[governance.md](./governance.md)** — how these docs evolve
- **[security.md](./security.md)** — security posture for the project and ecosystem

## Canonical links (recommended)

When referencing these docs from the code repos, link to:
- `manifesto.md`
- `principles.md`
- `what-pathops-is-not.md`

So the project stays aligned over time.
